# Deployment — Docker + Coolify

## Coolify setup

- **Coolify version:** v4.1.2
- **Server:** localhost (192.168.1.26)
- **Build pack:** Docker Compose
- **Repo:** `daniele-boerio/movie-recommender`, branch `main`
- **Domain frontend:** `https://movies.spassocasa.it` (routed by Traefik)
- **Domain backend:** NONE — backend is internal only, not exposed
- **Compose location:** `/docker-compose.yml`

## Environment variables (set in Coolify UI)

| Variable | Where | Notes |
|----------|-------|-------|
| `TMDB_API_KEY` | Coolify env vars | Required. Passed to backend container |
| `SERVICE_FQDN_FRONTEND` | Auto-generated by Coolify | Magic var for Traefik routing |
| `DB_PATH` | Set in compose | `/data/watchnext.db` on the Docker volume |

## Docker architecture

### frontend (Dockerfile: `frontend/Dockerfile`)
Multi-stage build:
1. **Stage 1 (node:20-alpine):** `npm install` → `npm run build` → produces `/app/dist`
2. **Stage 2 (nginx:alpine):** copies `dist/` to `/usr/share/nginx/html`, uses custom `nginx.conf`

### backend (Dockerfile: `backend/Dockerfile`)
Single stage, `python:3.12-slim`. Installs deps, runs uvicorn on `:8000`. SQLite DB at `/data/watchnext.db` via mounted volume.

### Nginx reverse proxy (`frontend/nginx.conf`)
```
location /          → serves static React files (try_files with SPA fallback)
location /api/      → proxy_pass http://backend:8000
```

The `backend` hostname resolves via Docker Compose internal DNS (both services share a network).

### Volume
`watchnext-data` — mounted at `/data` in the backend container. Holds `watchnext.db`. Persists across redeploys.

⚠️ When deleting the Docker Compose resource in Coolify, choose "Keep Volumes" to preserve the database.

## docker-compose.yml rules for Coolify

- Do NOT expose ports with `ports:` — Traefik handles external routing
- Use `SERVICE_FQDN_*` magic vars for auto-generated domains
- Environment variables set in Coolify UI override those in the compose file
- `depends_on` is respected for startup ordering
- `healthcheck` is respected — Coolify waits for health before marking as running
- Named volumes survive redeploys; bind mounts may not

## Common deployment tasks

### Rebuild and redeploy
Push to `main` branch. If Coolify webhooks are configured, it auto-deploys. Otherwise, click "Redeploy" in the Coolify UI.

### View logs
Coolify UI → Logs tab, or:
```bash
docker logs -f <container-name>
```

### Access backend shell
Coolify UI → Terminal tab, select the backend service. Or:
```bash
docker exec -it <backend-container> bash
```

### Inspect SQLite database
```bash
docker exec -it <backend-container> python -c "
import sqlite3
conn = sqlite3.connect('/data/watchnext.db')
for row in conn.execute('SELECT tmdb_id, title, rating FROM watched'):
    print(row)
conn.close()
"
```

### Reset database
```bash
docker exec -it <backend-container> rm /data/watchnext.db
# Then restart the backend — init_db() recreates the table
```

### Test backend without frontend
```bash
# From the server or any machine on the network:
curl http://192.168.1.26:8000/api/trending
# Won't work if backend port isn't exposed — use Coolify Terminal instead
```

## Troubleshooting

| Problem | Cause | Fix |
|---------|-------|-----|
| `npm ci` fails during build | No `package-lock.json` in repo | Use `npm install` in Dockerfile (already fixed) |
| DNS validation failed for backend | Backend domain set in Coolify | Remove backend domain — it's internal only |
| Frontend loads but API returns 502 | Backend container not ready yet | Check backend logs; ensure `TMDB_API_KEY` is set |
| TMDB returns 401 | Invalid or missing API key | Verify `TMDB_API_KEY` in Coolify env vars |
| Database empty after redeploy | Volume not mounted or got deleted | Check `watchnext-data` volume exists |
| Frontend shows blank page | Vite build failed silently | Check frontend build logs in Coolify Deployments tab |
| `proxy_pass` returns 502 | Backend hostname wrong in nginx.conf | Must be `backend` (the compose service name) |
